## Elastic BeanStalk 실습 1
- AWS메뉴 -> Elastic BeanStalk
- 애플리케이션 생성 선택
  - 1. 환경 구성
    - 웹 서버 환경 선택
  - 애플리케이션 정보
    - 애플리 케이션 이름 작성
  - 환경 정보
    - 환경 이름 작성
    - 도메인 이름 작성
  - 플랫폼
    - 플랫폼 선택 : PHP
    - 플랫폼 프랜치 선택
    - 플랫폼 버전 선택
  - 애플리케이션 코드
    - 샘플 애플리케이션
    - 코드 업로드 < 선택
      - 버전 레비을 작성 : v0
      - 로컬 파일 / S3 URL
  - 사전 설정
    - 구성 사전 설정
      - 단일 인스턴스 (프리 티어 사용 가능) 선택
  - 2. 서비스 액세스 구성
    - 만들어진 것이 있다면 기존 서비스 역할 사용 or 새로 만들것
    - 이전에는 AWS 에서 EC2인스턴스를 만들어 줬으나 이제는 보안 가이드라인에서 위배가 되어 자동으로 인스턴스 및 비즈니스 역할을 생성해 주지 않음
    - 그러므로 beanstalk를 사용하기 위해선 서비스 역할을 만들고 몇가지 추가해야 하는 것이 있음
      - AWSElasticBeanstalkWebTier
      - AWSElasticBeanstalkWorkerTier
      - AWSElasticBeanstalkMulticontainerDocker
    - IAM 역할 만들어서 권한에 위에 3개를 넣어둘것
    - EC2 키 페어 
      - 만들어진 것이 있다면 사용 or 새로 만들것
    - EC2 인스턴스 프로파일
      - 만들어진 것이 있다면 사용 or 새로 만들것
  - 3 ~ 5 단계는 선택사항이라 건너뜀
  - 제출 버튼 클릭

## Elastic BeanStalk 실습 2
- AWS메뉴 -> Elastic BeanStalk > 애플리케이션 선택 > 애플리케이션 버전
- 업로드 선택
  - 버전 레이블 입력 : V2
  - 설명 입력
- 환경 메뉴 -> 구성
  - 업데이트, 모니터링 및 로깅 구성 선택
  - 중간 쯤에 롤링 업데이트 및 배포
    - 배포 정책 : 한 번에 모두
- 어플리케이션 버전에서 V2 선택
  - 어플리케이션 버전 배포
  - 환경 선택
