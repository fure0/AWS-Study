## RDS의 MULTI-AZ 기능으로 단일 장애지점은 없앤 거 같은데, ELB도 안딜 장애 지점 같은데도 저건 어떻게 이중화 하죠?
- ELB는 단일 장애 지점이 아님
- ELB는 우리가 관리 하는게 아니고 AWS가 관리하는 서비스 이기 때문에 내부적으로 이중화가 되어있음
- 실제 우리가 ELB 만들 때 가용영역 최소2개를 선택하는 부분이 있었음
- 가용영역 마다 하나씩 설치가 됨

## 쇼핑몰이다 보니 제품 홍보용 이미지, 동영상들 많이 사용하는데, 이 파일들을 WEB 서버에 보관 했더니 문제가 생기기 시작했어요.
- 처음에는 이미지랑 동영상이 몇 개 안돼서 WEB 서버 다 똑같은 이미지, 동영상을 복사해서 운영함
- 이러니까 고객이 많이 들어 올수록 WEB 서버가 여러 개 생기면 다 똑같은 파일들을 저장하는데 다 겹처서 저장하고 있는 것임
- 쓸데업이 스토리지 비용들만 낭비되는거 같음
- 어디 중앙 집중식으로 이미지랑 동영상 저장해 놓고 사용자에게 보내줄 수 없을까?
- 한번 파일을 업로드하고 인터넷을 통해 많은 사람이 다운로드 받을 수 있도록 하는 스토리지

## S3라는 서비스는 웹하드, 구글 드라이브 같은 서비스래요. 파일을 올려놓고 인터넷을 통해 많은 사람이 받아 갈 수 있에 하는 서비스
- WEB 서버에 있는 동영상과 이미지들을 S3로 옮기고 관련 있는 주소들을 변경해줌.
- WEB 서버마다 EBS라는 SSD가 붙어 있었는데, 이 SSD들이 더이상 똑같은 파일을 다 가지고 있지 않아도 되겠음
- 이제 WEB 서버의 부담도 조금 줄어 들 듯함
- S3에는 스태틱 웹사이트 호스팅 기능이 있어서 웹서버의 기능을 대체해서 사용할 수도 있다고 함
