## 앗! 추적해보니 제가 50개를 만들었다고 나오네요. 뭐지?
- 인스턴스 50개를 만든 적이 없음.
- Root 계정이 실행된 IP를 보니 다른 지역에서 실행된거 같음.
- 아무래도 해킹당했나봄
- Root 계정 비밀번호를 변경하고 인스턴스들을 지움
- 보니까 인스턴스들이 비트코인을 캐고 있었음
- 이번 달 AWS 비용 어마어마하게 나오겠음
- 급해서 PC방에서 AWS 로그인한 적이 있는데 그때 털렸나 봄

## 쇼핑몰에 문제가 있는지 S3의 데이터와 DB가 지워진 건 아닌지 확인해 봅니다.
- 다행히 S3의 데이터와 DB가 지워진 것도 아니고, 다른 것도 아무것도 안해음
- 그냥 인스턴스만 50개 만들어서 비트코인만 채굴 한거 같음

## 다시 고민을 합니다.
- 어떻게 해야 이런 어마어마한 사고가 다시 발생하지 않을까?
- 노트북은 나 밖에 사용하지 않기 때문에, 노트북에서만 AWS에 로그인 하면 안전할까?
- 그런데 AWS는 직원들도 같이 사용하는데 직원들이 실수를 하면 어떻게 하나?
- 찾아보니 AWS는 사용자를 관리 하기위한 서비스로 IAM을 제공함

## AWS는 IAM을 통해 사용자들의 권한을 관리 할 수 있습니다. IAM 잘 몰라서 안 쓰고 있었는데, 권한 관리 어떻게 하나 찾아봐야 겠네요
- IAM이 존재 한다는건 나도 들어서 알고 있음
- 단지 관리 방법과 권한주는 것을 잘 몰라서. 너무 복잡함. 그래서 Root로 작업하고 있던거임
- AWS의 IAM 모법 사례를 찾아봄
- Root 권한은 꼭 필요할때 말고는, 사용하지 말라고 나와 있음
- 그리고 MFA(Multi-factor authentication)를 설정해 one-time패스워드를 걸어서 사용 하라고 함
- 그리고 Root로 Admin 사용자를 만들어서 AdministratorAccess 권한을 줘서 앞으로는 Admin으로 다른 사용자들을 만들어서 사용 하라고 함
