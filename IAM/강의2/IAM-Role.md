  - Management Consolt
  - Terminal CLI

- ManageMent Console은 Username/PW를 가지고 인터넷으로 로그인
- CLS, SDK를 사용하려면 Access key랑 Secret Access key를 사용해야 되서 그걸 코드에 넣어서 사용해야 한다고 함

## 다시 고민을 합니다.
- 이렇게 사고로 키가 노출되는건 어떻게 막을 수 있을까?
- SDK나 CLI를 사용할때 키는 필수인거 같은데
- 노트북은 나만 쓰니까 괜찮다고 해도, SDK를 사용할때는 내 코드랑 같이 키가 인스턴스에 저장이 될텐데

## IAM의 ROLE을 사용하면 ACCESS KEY, SECRET ACCESS KEY에 대한 보안을 강화 할 수 있습니다.
- IAM의 Role(역할)이라는 기능이 있어서 AWS 서비스들 서로 호출할 때의 권한은 이걸로 하면 Key가 필요없다고 함
- Role(역할)은 여러 기능이 있는데, 우리는 여러 기능중에 하나인 서비스들 끼리의 권한을 Key 없이 주는 방법을 사용해 볼거임
- Role(역할)을 만들어서 권한을 주고 EC2에 적용 해주고, 코드에 들어가 있는 key들을 다 제거해 줄거임
- 이렇게 하면 AWS 서비스가 해킹 당해서 Key가 노출되는 일은 없을거임
